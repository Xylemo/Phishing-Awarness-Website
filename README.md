## About

Phishing is one of the most prevalent cyberattacks targeting everyday users. This project aims to close the awareness gap with an interactive, beginner-friendly website that:

- Explains **what phishing is** and how attackers exploit human psychology
- Provides **real-world examples** of phishing emails, links, and spoofed pages
- Builds the instincts users need to **identify suspicious content** before it's too late

No cybersecurity background needed — if you can read an email, this site is for you.

## Viewing the Site Locally

1. Open `index.html` in the editor
2. Right-click anywhere in the file
3. Select **"Open with Live Server"**

Your browser will open the site at `http://127.0.0.1:5500` and live-reload on every save.

## Submitting Your Work

Submit changes as a **Pull Request (PR)** using VS Code's built-in Source Control panel. A PR lets the team review your work before it's merged — keeping the main branch stable.

> [!IMPORTANT]
> Never push directly to `main`. Always open a Pull Request so your changes can be reviewed first.

### Step by Step

1. Click the **Source Control icon** in the left sidebar (or press `Ctrl+Shift+G`)
2. Hover over each changed file and click **`+`** to **stage** it
3. Type a short, descriptive **commit message** at the top

   > ✅ `"Add quiz question about URL spoofing"`  
   > ❌ `"changes"` or `"update"`

4. Click the **checkmark ✓** to commit
5. Click **"Publish Branch"** or **"Push"** to upload to GitHub
6. Click **"Sync Changes"** if prompted
7. Open the repository on GitHub — you'll see a prompt to **"Create Pull Request"**
8. Add a title and description, then submit for review

## Staying Up to Date

Always pull the latest changes before starting work. This keeps your local copy current and prevents merge conflicts — situations where two people edited the same lines differently.

```bash
# Open Terminal → New Terminal (Ctrl+`)
git pull
```

<img width="1301" height="242" alt="VS Code integrated terminal with 'git pull' command completing successfully" src="https://github.com/user-attachments/assets/a1981195-afc3-4192-b1a1-c3aed4ca27f0" />

*▲ Run `git pull` in the integrated terminal at the start of every work session.*

> [!IMPORTANT]
> Skipping `git pull` is the #1 cause of merge conflicts in team projects. Make it a habit.
